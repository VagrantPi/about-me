自我介紹

我畢業於屏東大學資訊工程學系，在學期間程式相關課程都有不錯的成績，如果教授有出加分作業，我通常都是勇於挑戰的那個， 軟體工程課時，第一次多人協作，不過幾乎都是各做各的，而我只能畫畫UML，幫幫小忙，這時讓我開始思考，業界多人合作時會是怎樣呢? 應該不會也是這樣吧?

正當我開始迷惘時，因緣際會下開始接觸了些技術社群，除了開拓視野外，我也找尋並學習吸收非校內的資訊，更於 SITCON 2016 當上了講師， 讓我更確信了，這條道路是值得走的

在社群朋友的介紹下，很幸運地來到台中的創科資訊實習，公司有在幫別人做企業培訓的課程，一位被帶去當打共筆的助教， 我也因此多了更多的學習機會，除此之外公司也會定期舉辦 meetup，並且在暑假時成了『HelloJS：暑期WEB 工程師訓練營』 的主辦人，讓我學到了很多

實習期間，被大量灌輸了很多工程師必備姿勢(XDD)，比如說:
  • 寫 code 要嚴警點，不然到時踩到地雷的可能是你自己
  • 自我提升、進修，不只是在幫住團隊，也是在幫助自己(你不可能一輩子都 寫同樣的 code or 不失業)
  • 當你有"意識"到程式的品質是重要的，才能改善程式的品質
  • ....

除此之外了學到很多 tool 應用: Sequelize、Lodash、寫測試、TDD、Scrum、SQL、PHP....

接下來，我希望能繼續學習成長，並且成為大大，然後回到社群幫助更多資訊小幼苗
